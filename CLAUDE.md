# CLAUDE.md — standing rules for every session in this repo

Tutti is a map-first directory helping LA parents find a youth orchestra for their
kid, and players find auditions. It is curated by Milai Walker (professional
cellist and music teacher, non-programmer, the product owner and sole approver).
Its entire value is **trust**. These rules are not preferences; they are the product.

## The honesty contract (never weaken it)

- A fact is either **verified** `{status:"verified", value, source, asOf}` — value
  shown, source cited — or **unverified** `{status:"unverified"[, source]}` — shown
  as "Check their site," never a value. `not_applicable` hides the row. The optional
  `source` on an unverified fact is a **pointer** (the page to check), and may only
  be a page a human actually consulted.
- **Never invent or infer prices, dates, tuition, audition windows, or any org
  fact.** Not in code, not in copy, not in test data destined for main.
- Only `listing_status = 'published'` rows are publicly visible. The database
  enforces this (RLS) and enforces the verified/unverified shape and the
  greater-LA scope (CHECK constraints). Do not loosen migrations to "make
  something work."
- Ratings stay `null` — no trustworthy source exists. Scope stays greater Los
  Angeles.
- Adding, removing, or changing **listing facts is not a Claude Code task.**
  Curation (web research, citations, judgment) happens in Milai's Claude.ai
  project with her approval. If a task seems to require new org facts, stop and
  ask Milai.

## Fixed contracts

- `data.json` is the source of truth for listings; `schema.json` / `SCHEMA.md`
  define its shape; the seed and the HTML's embedded bootstrap are **generated
  from it** and must never drift. `HANDOFF.md` orients; keep docs updated in the
  same commit as the change they describe.
- Design tokens are fixed: type colors (youth = brass/gold, professional =
  garnet/red, community = teal, collegiate = slate/grey), Fraunces for display,
  Mulish for body, musical note-shaped pins, sheet-music staff lines behind panel
  headers. No new colors/fonts and no data-field renames without flagging to
  Milai first, with the reason.
- Copy is written for a parent, not a musician: plain words, named by what the
  parent does ("Save", "Visit website", "Player age").

## Engineering rules

- Single-file app (`tutti-orchestra-finder.html`) + `config.js` + Supabase
  (`supabase/` migrations, seed generated by scripts). No frameworks, no build
  step. **No localStorage/sessionStorage** — in-memory plus Supabase only.
- Quality bar: works on phones; keyboard users can always see where they are;
  respects reduced-motion; a styling change must never reveal hidden sections.
- Migrations are append-only: new numbered files, never edit an applied one.
- Secrets live only in `.env` (gitignored). Never commit, print, or echo them.
  Never overwrite values in the user's `config.js`; add keys additively.
- **Verify the base before editing.** Read the actual files on the current
  branch; never patch from memory of an older snapshot. (A stale-snapshot edit
  once deleted the whole auth layer. Recovery exists — `git show <hash>:file` —
  but don't need it.)
- Work on feature branches (`phase4/<task>`), self-review with /code-review,
  then present Milai a plain-words summary with the done-when evidence. She
  merges (or says "merge it").

## Escalate to Milai instead of guessing

Business choices (money, providers, scope), schema shape changes, any copy that
changes a promise to parents, anything touching listing facts, and anything
where two readings of the task are plausible. One clear question beats a wrong
build.
